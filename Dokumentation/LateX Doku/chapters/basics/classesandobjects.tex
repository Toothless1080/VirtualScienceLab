\section{Klassen und Objekte}

Python ist wie Java eine objektorientierte Programmiersprache.
Das bedeutet, dass in Python fast alles aus Objekten und Klassen besteht.
Klassen sind Vorlagen, aus denen Objekte generiert werden können.
Dabei enthält die Klasse je nach Verwendungszweck Variablen und Methoden.
Ein Beispiel hierfür wäre die Klasse \lstinline{Kreis}. Jeder Kreis besitzt einen Radius, jedoch besitzt nicht jeder Kreis den selben.

Wie in Abschnitt \ref{basicdatatypes:sec:ElementareDatentypen} beschrieben wird der Datentyp einer Variable anhand der Daten, auf die sie referenziert, bestimmt. Auch bei elementaren Datentypen entspricht dies einer Zuordnung zu einer Klasse.
Dies lässt sich durch folgendes Listing nachvollziehen:
\begin{lstlisting}[language=Python,
label={classesandobjects:additionasmethod}]
#Die Variable i referenziert die 5
i = 5
#Da 5 einem Integer entspricht wird i dieser Klasse zugeordnet
print(type(i))
#Addition mittels Operand
print(i + 3)

#Variable j referenziert die 4
j = 4
#Variable j wird der Klasse Integer zugeordnet
print(type(j))
#Addition mittels Methode
print(j.__add__(7))
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python, label={classesandobjects:subsection:createclassesandobjects:outputtypes}]
<class 'int'>
8
<class 'int'>
11
\end{lstlisting}
Dadurch dass die Variablen i und j der Klasse Integer zugeordnet werden ist es möglich mittels Operand oder Methode eine Addition durchzuführen.

\subsection{Klassen und Objekte erstellen}

Um dies Anhand eines Python Programms zu verdeutlichen, wird eine neue Klasse erstellt.
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:createclassesandobjects:createclass}]
# Erstellen einer Klasse
class Kreis:
    radius = 1
\end{lstlisting}
Mithilfe der erstellten Klasse, können nun verschiedene Objekte erstellt werden, welche die Variablen und Methoden der Klasse beinhalten.
\begin{lstlisting}[language=Python, label={classesandobjects:subsection:createclassesandobjects:studentobject}]
class Kreis:
    radius = 1
    
kreis1 = Kreis()
print(kreis1.radius)
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python, label={classesandobjects:subsection:createclassesandobjects:outputname}]
1
\end{lstlisting}
Die Variablen der Objekte sind zunächst gleich mit denen der Klasse, aus der diese erstellt wurden.
Allerdings sind die Variablen des erstellten Objekts unabhängig von denen der Klasse. Das bedeutet, dass diese auch unabhängig für jedes einzelne Objekt geändert werden können. Die im obigen Beispiel verwendete Klasse ist in realen Anwendungen  nicht verwendbar, da die Attribute des Objekts von Anfang an festgelegt wurden. Um einen dynamischen Ansatz nutzen zu können, sollte man wie im folgenden Beispiel vorgehen.
\begin{lstlisting}[language=Python, label={classesandobjects:subsection:createclassesandobjects:selfinit}]
class Kreis:
    def __init__(self, radius):
        self.radius = radius

kreis1 = Kreis(3)
kreis2 = Kreis(5)
print(kreis1.radius)
print(kreis2.radius)
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:initclass:outputtwonames}]
3
5
\end{lstlisting}

\subsection{Die \textunderscore \textunderscore init\textunderscore \textunderscore () Methode}
Jede Klasse hat eine \lstinline{__init__} Methode, die immer ausgeführt wird, wenn die Klasse initiiert und ausgeführt wird. Diese Methode wird verwendet um den Variablen des Objektes einen Wert zu geben.
Hierbei ist der \lstinline{self} Parameter notwendig um die Klasseninstanz selbst, somit das generierte Objekt, zu referenzieren und auf dessen Variablen zugreifen zu können. Dieser Parameter ist also notwendig, muss aber nicht self genannt werden, sondern kann einen beliebigen Namen haben. Er wird als erster Parameter bei jeder Methode angegeben.
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:initclass:initclass}]
class Kreis:
    def __init__(self, radius):
        self.radius = radius

    def getRadius(self):
        print(self.radius)

kreis1 = Kreis(3)
kreis2 = Kreis(5)
print(kreis1.radius)
print(kreis2.radius)
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:initclass:outputnames2}]
3
5
\end{lstlisting}
Objekte enthalten die aus den Klassen übernommenen Methoden. Diese Methoden gehören jetzt zu dem Objekt. Um Parameter zu modifizieren oder zu löschen, können folgende Befehle verwendet werden.
\begin{itemize}
	\item  \lstinline{objektname.parameter = neuer Wert (modifizieren)}
	\item \lstinline{del objektname.parameter (löschen)}
\end{itemize}

\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:initclass:delobjektname}]
class Kreis:
    def __init__(self, radius):
        self.radius = radius

    def getRadius(self):
        print(self.radius)

kreis1 = Kreis(3)
kreis2 = Kreis(5)
kreis1.radius = 3
kreis1.getRadius()
kreis2.getRadius()
del kreis2.radius
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:initclass:outputthreenames}]
3
5
\end{lstlisting}

\subsection{Vererbung}
Ein weiterer wichtiger Aspekt in Python, ist die Vererbung und Ergänzung einer Klasse.
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:heredity:heredity}]
class Kreis:
    def __init__(self, radius):
        self.radius = radius

    def getRadius(self):
        return self.radius

class Farbe(Kreis):
    def __init__(self, radius, farbe):
        Kreis.__init__(self, radius)
        self.farbe = farbe;

    def getRadius(self):
        print(str(Kreis.getRadius(self)) + ", " + self.farbe)

kreis1 = Farbe(3,"rot")
kreis2 = Farbe(5, "gelb")
kreis1.getRadius()
kreis2.getRadius()
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:heredity:heredityoutput}]
3, rot
5, gelb
\end{lstlisting}
Im Beispiel \ref{classesandobjects:subsection:heredity:heredity} wurde die Klasse Kreis durch die Klasse Farbe erweitert. Dies wird durch das Hinzufügen eines weiteren Attributes (bspw. farbe) realisiert. 
In dieser Klasse wurde die Methode  \lstinline{getRadius()} abgeändert. Die Klasse, von der geerbt wird, kann entweder mit dem Klassennamen  \lstinline{Kreis} oder mit  \lstinline{super} referenziert werden. 
Trotz der Änderung der Methode  \lstinline{getRadius()} in der Klasse  \lstinline{Farbe}, behält die Klasse  \lstinline{Kreis} ihre Methoden.

Innerhalb der Klasse Farbe ist es somit möglich über \lstinline{Kreis.getRadius()} oder mittels \lstinline{super().getRadius()} auf die überschriebene Methode \lstinline{getRadius()} der Klasse Kreis zuzugreifen. 
Da die Abgeleitete Klasse Farbe eigene Attribute besitzt werden diese über die \lstinline{__init__} Methode initialisiert. Hierbei wird die \lstinline{__init__} Methode der Klasse Kreis überschrieben und somit, beim erzeugen eines Objektes des Typs Farbe, nicht mehr automatisch aufgerufen. 
Damit das geerbte Attribut \lstinline{radius} initialisiert und verfügbar ist, muss die \lstinline{__init__} Methode der Klasse Kreis explizit aufgerufen werden.

\subsection{Die \textunderscore \textunderscore del\textunderscore \textunderscore () Methode}
Daraufhin bleibt zu erwähnen, dass auch komplette Objekte löschbar sind. Dies ist mit dem Befehl  \lstinline{del Objektname} möglich. Im nachfolgenden Listing wird eine Fehlermeldung ausgegeben, da versucht wird, auf das gelöschte Objekt zuzugreifen.
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:heredity:heredity}]
class Kreis:
    def __init__(self, radius):
        self.radius = radius

    def getRadius(self):
        print(self.radius)

kreis1 = Kreis(3)
kreis2 = Kreis(5)
kreis1.radius = 6
kreis1.getRadius()
kreis2.getRadius()
del kreis2.radius
del kreis2
kreis2.getRadius()
\end{lstlisting}
Fehlermeldung beim Ausführen des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:heredity:errormessage}]
Traceback (most recent call last):
6
File"C:/users/Patrick/Desktop/python/objekte
_undklassen.py", line 15, in <module>
5
kreis2.getRadius()
NameError: name 'kreis2' is not defined
\end{lstlisting}
Es ist möglich dass mehrere Referenzen auf ein Objekt zeigen. Erst sobald alle Referenzen auf ein Objekt mittels \lstinline{del} gelöscht wurden wird das Objekt selbst gelöscht.
Die Methode \lstinline{__del__} wird, sofern sie implementiert ist, aufgerufen sobald ein Objekt gelöscht wird. Zu diesem Zeitpunkt sind die Attribute nicht mehr verfügbar, weshalb in der \lstinline{__del__} Methode nicht mehr auf sie zugegriffen werden kann. Sehen wir uns hierzu folgendes Listing an:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:delmethod:notdefined}]
class A():
    def __init__(self, name):
        self.name = name
        print(name + " wurde erzeugt!")

    def __del__(self):
        print("Objekt der Klasse A wurde gelöscht")

class B():
    def __init__(self, name):
        self.name = name
        print(name + " wurde erzeugt!")

    def __del__(self):
        print("Objekt der Klasse " + name + " wurde gelöscht")

x = A("A")
y = B("B")

#Z erhält die gleiche Referenz wie x
#Beide zeigen somit auf ein Objekt der Klasse A
z = x

print("Löschen von x der Klasse A")
del x
print("Löschen von z der Klasse A")
del z
print("Löschen von y der Klasse B")
del y
\end{lstlisting}
Fehlermeldung beim Ausführen des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:delmethod:notdefinedoutput}]
A wurde erzeugt!
B wurde erzeugt!
Löschen von x der Klasse A
Löschen von z der Klasse A
Objekt wurde gelöscht
Löschen von y der Klasse B
Exception ignored in:
<bound method B.__del__ of <__main__.B object at 0x7f6813e3a898>>
...
NameError: name 'name' is not defined
\end{lstlisting}
Anfänglich wird ein Objekt der Klasse A erzeugt und durch x referenziert. Ebenso eines der Klasse B und durch y referenziert.
Durch \lstinline{z = x} gibt es Zwei Referenzen auf das Objekt der Klasse A. Erst sobald beide gelöscht sind wird der entsprechende Text der \lstinline{__del__} Methode ausgegeben. In der Letzten Zeile des Listings wird y gelöscht, was jedoch zu einem Fehler führt, da in der \lstinline{__del__} Methode auf ein Attribut zugegriffen wird welches nicht mehr verfügbar ist.

Bezüglich der Vererbung verhält es sich bei der \lstinline{__del__} Methode wie bei der \lstinline{__init__} Methode, sie überschreibt die Methode der Klasse von der geerbt wird.

\subsection{Klassenattribute}
Im Gegensatz zu den bis jetzt betrachteten Instanzattributen die jedes erzeugte Objekt für sich hat, greifen die Objekte einer Klasse gemeinsam auf die Klassenattribute zu. Hierdurch betrifft eine Änderung der Klassenattribute jedes Objekt der Klasse. Es ist auch möglich Klassenattribute zu verwenden ohne ein Objekt der Klasse erzeugt zu haben. Folgendes Listing soll zeigen wie ein Klassenattribut verwendet werden kann:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:classattributes:counter}]
class A:
    counter = 0

    def __init__(self):
        A.counter += 1

    def __del__(self):
        A.counter -= 1

x = A();
print("Anzahl Instanzen von A:" + str(A.counter))
y = A();

#Klassenattribute sind auch über die Klasseninstanz erreichbar
print("Anzahl Instanzen von A:" + str(y.counter))

del y
print("Anzahl Instanzen von A:" + str(A.counter))
del x
print("Anzahl Instanzen von A:" + str(A.counter))
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:classattributes:counteroutput}]
Anzahl Instanzen von A:1
Anzahl Instanzen von A:2
Anzahl Instanzen von A:1
Anzahl Instanzen von A:0
\end{lstlisting}
Klassenattribute werden in der Klasse ohne \lstinline{self} initialisiert.
In diesem Beispiel ist dies \lstinline{counter}. Dieser speichert die Anzahl der von der Klasse A erzeugten Objekte.
Da jedes Objekt beim Erzeugen die \lstinline{__init__} 
bzw. beim Löschen die \lstinline{__del__} Methode aufruft, kann hierdurch das objektübergreifende Attribut 
\lstinline{counter} entsprechend angepasst werden.

\subsection{Statische Methoden}
Statische Methoden werden nicht über Objekte einer Klasse aufgerufen, sondern direkt über die entsprechende Klasse. Sie können dazu benutzt werden Operationen bereitzustellen die unabhängig von den einzelnen Objekten sind. Sie werden als Funktion definiert und dann mittels \lstinline{staticmethod} an eine Klasse gebunden.
In Python gibt es keine Funktionsüberladung. Dies bedeutet sobald Funktionen mit dem gleichen Namen im selben Namensraum definiert werden würde dies zu Fehlern führen. Dies gilt auch für Methoden innerhalb einer Klasse. Nun ist es mittels statischen Methoden möglich Funktionen zur Objekterzeugung an eine Klasse zu binden. Nachfolgendes Listing soll dies verdeutlichen:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:staticmethods:preinitfunction}]
def standardType():
    return A("default")

class A:
    def __init__(self, type):
        self.type = type

    def getType(self):
        print(self.type)

#Bindet die Funktion als statische Methode an die Klasse
    standardType = staticmethod(standardType)

x = A("extra")
y = A.standardType()

x.getType()
y.getType()
\end{lstlisting}
Ausgabe des Programmcodes:
\begin{lstlisting}[language=Python,
label={classesandobjects:subsection:staticmethods:preinitfunctionoutput}]
extra
default
\end{lstlisting}
Mit Hilfe der Funktion \lstinline{standardType()}, welche an die Klasse A gebunden ist, kann zur Erzeugung des Objektes ein Parameter Wert für die \lstinline{__init__} Methode festgelegt werden.
